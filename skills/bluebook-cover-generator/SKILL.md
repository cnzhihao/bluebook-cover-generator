# 蓝皮书封面生成器

通过 `opencli bluebook-cover-generator generate` 生成正式研究报告封面，并把 PNG 保存到本地。

## 前置检查

首次调用前执行：

```bash
opencli bluebook-cover-generator --help
opencli doctor
```

如果插件未安装，指导用户从项目仓库安装。不要退化为逐项点击网页或截图裁切。

## 内容准备

用户只给出主题或报告文件时，主动提炼以下内容：

- 主标题：简洁、正式，保留报告类型，例如“人工智能产业发展蓝皮书”。
- 副标题：一句话说明研究范围、年份或核心问题。
- 英文标题：忠实翻译主标题；不确定时可省略。
- 系列名和关键词：有明确上下文才填写，关键词最多四个。

机构、作者和 Logo 无法可靠推断时再询问用户；不能编造署名。用户未要求署名时可以留空。

## 样式选择

- 默认正式蓝皮书：`blueGold`
- 高端商务或咨询报告：`blackGold`
- 智库、生态、可持续研究：`green`
- 学术、教育或需要浅色背景：`light`

除非用户明确要求，不主动覆盖模板颜色和透明度。默认使用 2× PNG。

## 调用

始终使用 `-f json`，并为包含空格的内容加引号：

```bash
opencli bluebook-cover-generator generate '<主标题>' \
  --subtitle '<副标题>' \
  --englishTitle '<英文标题>' \
  --series '<系列名称>' \
  --tags '<关键词1, 关键词2>' \
  --org '<机构>' \
  --author '<作者>' \
  --date '<发布日期>' \
  --template blueGold \
  --output '<目标路径>.png' \
  -f json
```

有 Logo 时加 `--logo '<本地图片路径>'`。只传用户提供或可可靠推断的字段。

## 完成检查

解析 JSON 返回值，确认：

- `status` 是 `ok`
- 返回的绝对文件路径存在
- 2× 默认尺寸是 `1600×2262`

最后向用户报告 PNG 的绝对路径、尺寸和使用的模板。
